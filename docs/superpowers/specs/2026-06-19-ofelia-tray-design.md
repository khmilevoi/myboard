# Виджет «Лоток Офелии» — новые фичи

- **Дата:** 2026-06-19
- **Статус:** утверждён к реализации
- **Виджет:** `client/widgets/ofelia-poop-duty/`
- **Референс дизайна:** [`assets/2026-06-19-ofelia-tray.html`](./assets/2026-06-19-ofelia-tray.html)

## 1. Обзор

Виджет «Лоток Офелии» показывает, чья сегодня очередь убирать за собакой, и ведёт
учёт «долга» (кто кому должен дни). Сейчас виджет умеет немного: бинарный режим
`small`/`large`, экран «сегодня/завтра», модель с ротацией и счётчиком долга.

Новый дизайн расширяет виджет до полноценного флоу:

1. **Прогрессивные тиры** по размеру ячейки сетки (`tiny`/`compact`/`standard`/`large`)
   плюс отдельный `fullscreen`.
2. **Подтверждение уборки** + действия **«В долг»** и **«Простить»** с откатом.
3. **История** — per-week лог всех действий с серверным штампом IP.
4. **Комментарии** — per-week тред с ручным выбором автора.
5. **Неделя-календарь** с подсветкой дней гашения долга и **баланс долга**.
6. **Визуальный редизайн** под дизайн-референс.

Спека намеренно разбита на **независимые фичи** (раздел 5): каждая берётся отдельным
планом и может вестись параллельно. Стыки между фичами зафиксированы как **контракты**
(раздел 4), чтобы фичи разрабатывались против интерфейса, а не друг против друга.

### Принятые решения (из брейншторминга)

- **Объём:** все фичи сразу (тиры, подтверждение, «в долг», неделя, баланс, история,
  комментарии).
- **Авторизации нет** — оба участника могут выполнять любые действия; ручной тоггл
  К/Л только атрибутирует автора комментария.
- **История и комментарии** хранятся per-week (ключ привязан к началу недели);
  переключение недели меняет и календарь, и историю, и комментарии.
- **IP** фиксируется на сервере и хранится целиком; в UI истории показываются только
  последние несколько символов лёгким шрифтом под именем. В комментариях IP хранится,
  но не отображается.
- **Большой долг (`> 7`):** мягкое визуальное предупреждение, без жёсткого потолка.
- **Напоминания/push:** вне скоупа.
- **Тир** считается из размера ячейки сетки `[w, h]` (юниты, не пиксели) по конфигу на
  тип виджета и прокидывается в виджет как тег. `fullscreen` — отдельный тир, его
  выставляет оверлей.

## 3. Доменная модель и семантика

### 3.1 Ростер и день

- Ростер хардкод: `Леша`, `Карина` (`DUTY_ROTATION`).
- «Сегодняшний дежурный» = `день.debt ?? день.duty`, где `duty` — ротация, а `debt` —
  проекция дня гашения долга (уже есть в `currentWeek()`).
- «Подтверждено сегодня» — **производное** состояние: в логе **текущей** недели есть
  событие `cleaned` или `went_into_debt` с датой = сегодня (Europe/Warsaw).

### 3.2 Действия

Все действия атрибутируются, авторизации нет. Эффект на долг и запись в историю:

| Действие | Эффект на долг | Событие истории |
|---|---|---|
| **Подтвердить уборку** («Какашки убраны») | если сегодня день гашения долга → долг дежурного `−1`; иначе без изменений | `cleaned` (actor = дежурный; `onBehalfOf` = кредитор, если гасил долг) |
| **В долг** | дежурному `+1` день; сегодня убирает другой | `went_into_debt` (actor = тот, кто убрал; `onBehalfOf` = дежурный) |
| **Простить** | долг должника `−1` | `forgiven` (actor = тот, кто простил) |
| **Откатить** (рядом с «подтверждено») | реверс последнего сегодняшнего события | удаление соответствующей записи |

- Счётчик долга нормализуется так, что минимум = 0 (как сейчас, `normalizeDebts`).
- Долг не уходит ниже 0.
- **Предупреждение:** при `debt[x] > DEBT_WARNING_THRESHOLD` (`= 7`) — визуальный акцент
  в чипах/балансе. Жёсткого потолка нет.

### 3.3 История (per-week)

- Ключ: `history:<weekStartISO>` в namespace `shared.server` виджета
  (полный ключ вида `w:t:ofelia-poop-duty:history:2026-06-15`).
- `weekStartISO` — понедельник недели (`getStartOfWeek`, Europe/Warsaw).
- Значение — JSON-массив `HistoryEvent` (см. контракт 4.3), упорядоченный по `ts`.
- Запись только через серверный **append** (раздел 4.2): сервер проставляет `id`, `ts`, `ip`.
- Чтение/реактивность — через существующий `withStorageKey` (SSE-подписка).
- Левая «сегодня»-панель читает лог **недели сегодня**; правая панель (календарь/история/
  комментарии) — лог **выбранной** недели.

### 3.4 Комментарии (per-week)

- Ключ: `comments:<weekStartISO>` (аналогично истории).
- Значение — JSON-массив `Comment` (контракт 4.4).
- Автор берётся из ручного тоггла К/Л. IP хранится (через тот же append), не отображается.

## 4. Контракты (швы между фичами)

Эти типы/интерфейсы фиксируются первыми; каждая фича реализуется против них.

### 4.1 Тир

```ts
// client/widgets/ofelia-poop-duty/... или общий в widget-host
export type WidgetTier = 'tiny' | 'compact' | 'standard' | 'large' | 'fullscreen'

// Пороги в юнитах сетки; тир = наибольший, у которого w >= minW && h >= minH.
// Пол — 'tiny'. 'fullscreen' сюда не входит (его выставляет оверлей).
export type TierThreshold = { minW: number; minH: number }
export type TierConfig = {
  tiny: TierThreshold
  compact: TierThreshold
  standard: TierThreshold
  large: TierThreshold
}

export function resolveTier(size: { w: number; h: number }, config: TierConfig): WidgetTier
```

`WidgetRuntimeProps` получает поле `tier: WidgetTier` (поле `mode` остаётся для
совместимости с другими виджетами).

### 4.2 Серверный append (storage)

```ts
// расширение StorageApi
append<T>(key: string, entry: T, options?: { cap?: number }): Promise<StorageError | void>
```

- Семантика: дописать `entry` в конец JSON-массива по ключу; если ключа нет — создать `[entry]`;
  при заданном `cap` — обрезать слева до `cap` последних.
- Сервер обогащает запись полями `id` (uuid), `ts` (epoch ms), `ip`
  (`x-forwarded-for` → первый, иначе `socket.remoteAddress`, хранится целиком).
- Новое значение публикуется в SSE как обычный `set` (подписчики получают весь массив).
- Атомарность: сервер сериализует append'ы **по ключу** in-process (один инстанс Node),
  чтобы параллельные записи не терялись. (Апгрейд до Lua/atomic — если появится
  горизонтальное масштабирование; вне скоупа.)
- HTTP: `POST /api/storage/:key/append`, тело `{ entry, cap? }`, ответ `204`.

### 4.3 Событие истории

```ts
export type Person = 'Леша' | 'Карина'
export type HistoryEventType = 'cleaned' | 'went_into_debt' | 'forgiven'

export type HistoryEvent = {
  id: string                 // сервер
  ts: number                 // сервер, epoch ms
  ip: string                 // сервер, хранится целиком; UI показывает хвост
  date: string               // ISO PlainDate дежурного дня
  type: HistoryEventType
  actor: Person              // кто выполнил/убрал
  onBehalfOf?: Person        // «за X» — когда покрыли чужой день
}

// то, что эмитит модель действий (без серверных полей):
export type HistoryEventDraft = Omit<HistoryEvent, 'id' | 'ts' | 'ip'>
```

### 4.4 Комментарий

```ts
export type Comment = {
  id: string                 // сервер
  ts: number                 // сервер
  ip?: string                // сервер, не отображается
  author: Person             // из тоггла К/Л
  text: string
}
export type CommentDraft = Pick<Comment, 'author' | 'text'>
```

### 4.5 Порт лога для модели действий (F3)

Модель действий не знает про storage/сервер — она зовёт инжектируемый порт:

```ts
export type HistoryPort = {
  append(event: HistoryEventDraft): Promise<void>
}
```

В проде порт обёрнут над `storage.shared.server.append(historyKey(today), draft)`;
в тестах — фейк. Так F3 не зависит от реализации F2/F4.

## 5. Фичи (независимые, каждая — свой план)

### F1 — Тир-инфраструктура (host)

- **Цель:** любой виджет получает `tier` исходя из размера ячейки; оверлей даёт `fullscreen`.
- **Скоуп:** `WidgetTier`, `TierConfig`, `resolveTier`; поле `tiers?: TierConfig` в `WidgetType`
  (+ `DEFAULT_TIERS`); `tier` в `WidgetRuntimeProps`; в `Board` — расчёт тира из layout-элемента
  инстанса (`{w,h}`) и проброс в `WidgetFrame`; в `FullscreenOverlay` — `tier="fullscreen"`.
- **Зависит от:** —
- **Контракты:** владеет 4.1.
- **Готово, когда:** виджет в карточке получает корректный тир по `[w,h]`, в оверлее —
  `fullscreen`; тесты `resolveTier` (границы) + Board/registry.
- **Тесты:** unit `resolveTier`; рендер Board прокидывает тир; overlay даёт `fullscreen`.

### F2 — Серверный append + IP (storage)

- **Цель:** атомарный per-key append в JSON-массив со штампом `id/ts/ip`.
- **Скоуп:** роут `POST /api/storage/:key/append` (`server/index.ts` + хендлер в
  `server/handlers.ts`, схема в `server/schemas.ts`, захват IP, in-process per-key lock,
  публикация change); метод `append` в `StorageApi` и его реализации
  (`server/http-storage.ts`, `client/dexie-storage.ts`, `test/fakes.ts`); хелпер мутации
  в `reatom-storage.ts` при необходимости.
- **Зависит от:** —
- **Контракты:** владеет 4.2.
- **Готово, когда:** `storage.append(key, entry)` дописывает на сервере с `ip/ts/id`,
  соблюдает `cap`, публикует SSE; e2e клиент→сервер.
- **Тесты:** server `handlers.test.ts` (append + cap + ip + публикация); client
  `http-storage.test.ts`/`dexie-storage.test.ts` (append).

### F3 — Действия и долг (Ofelia model)

- **Цель:** confirm / в-долг / простить / откат + предупреждение `>7`; эмит событий через порт.
- **Скоуп:** расширение `model/ofelia-duty.ts`: действия `confirmClean`, `goIntoDebt`,
  `forgive`, `undoToday`; вычисление «подтверждено сегодня»; константа
  `DEBT_WARNING_THRESHOLD`; приём `HistoryPort` как зависимости модели.
- **Зависит от:** контракт порта 4.5 (фейк в тестах), формы события 4.3.
- **Контракты:** владеет формой события 4.3 совместно с F4.
- **Готово, когда:** действия меняют долг согласно таблице 3.2 и эмитят корректные
  `HistoryEventDraft`; «подтверждено сегодня» выводится из лога.
- **Тесты:** unit с фейковым портом — каждое действие, откат, граница долга `0`,
  предупреждение `>7`, день гашения vs обычный день.

### F4 — История (Ofelia)

- **Цель:** per-week лог: чтение/подписка/аппенд + UI-список с хвостом IP.
- **Скоуп:** `model/ofelia-history.ts` (ключ `history:<weekStartISO>`, чтение через
  `withStorageKey`, `HistoryPort` поверх `storage.shared.server.append`); UI-часть
  `ui/parts/HistoryList.tsx` (дата, аватар, имя, бейдж «за X», хвост IP лёгким шрифтом).
- **Зависит от:** F2 (append), форма события 4.3 (F3).
- **Готово, когда:** лог выбранной недели рендерится и пополняется действиями; смена
  недели меняет лог.
- **Тесты:** model — ключ недели, маппинг событий; ui — рендер записей и хвоста IP.

### F5 — Комментарии (Ofelia)

- **Цель:** per-week тред с ручным выбором автора и вводом.
- **Скоуп:** `model/ofelia-comments.ts` (ключ `comments:<weekStartISO>`, чтение +
  append); UI `ui/parts/CommentThread.tsx` (список, тоггл автора К/Л, поле ввода, отправка
  по Enter/кнопке).
- **Зависит от:** F2 (append).
- **Контракты:** владеет 4.4.
- **Готово, когда:** комментарий недели добавляется и виден; автор берётся из тоггла.
- **Тесты:** model — ключ недели, append; ui — смена автора, отправка, очистка поля.

### F6 — Визуал и тиры UI

- **Цель:** перенести дизайн: 5 тиров и все состояния.
- **Скоуп:**
  - Токены (раздел 6): `--ofelia-k-bg/-fg`, `--ofelia-l-bg/-fg`, `--ofelia-ok-soft/-fg`,
    `--font-mono` (+ dark-варианты).
  - Части: `Avatar`, `DebtChips`, `WeekStrip` (календарь 7 дней с точками дней долга и
    подсветкой сегодня + легенда), `ActionButtons`.

    `FullscreenTier`; общий `RichLayout` (two-column: «сегодня» | неделя+история+комментарии),
    используемый `Large` и `Fullscreen`.
  - `OfeliaPoopDuty.tsx` — роутер по `tier`.
- **Зависит от:** F1 (`tier`); потребляет модели F3/F4/F5 по мере готовности.
- **Готово, когда:** каждый тир и состояние соответствуют референсу; mono-лейблы, аватары,

## 7. UI: тиры и состояния

- **tiny** — крупный аватар + имя сегодняшнего дежурного.
- **compact** — лейбл «Сегодня убирает», 3 иконки-действия (✓ подтвердить / ⏲ в долг /
  − простить), аватар+имя+«гасит долг», чипы долга `К n · Л 0`.
- **standard** — заголовок «Лоток Офелии», блок «сегодня», разделитель, строка «Долг» с
  чипами, пояснение, кнопки. Состояния: **A** долг, **B** подтверждено (плашка + откат),
  **C** долгов нет.
- **large** — крупная inline-карточка на доске: `RichLayout`.
- **fullscreen** — оверлей: `RichLayout` + аффордансы фуллскрина (закрытие). Отдельный тир,
  чтобы поведение можно было развести с `large`.

`RichLayout`: слева «Сегодня» (аватар, чип «гасит долг», подтвердить/откат,
«В долг»/«Простить», пояснение, «Баланс долга» К/Л с предупреждением при `>7`); справа —
навигация недель (`goToPrev/Next/CurrentWeek`) + `WeekStrip` + горизонтальный сплит
`HistoryList` | `CommentThread`. Навигация синхронно меняет календарь/историю/комментарии;
левая панель всегда показывает фактическое сегодня.

## 9. Вне скоупа / открытые мелочи

- Напоминания/push-уведомления.
- Жёсткий потолок долга (только мягкое предупреждение).
- Реальная авторизация/идентичность.
- Точные пороги `TierConfig` и `defaultSize` (сейчас `w:3,h:5`) — подобрать при F1/F6,
  чтобы дефолт попадал в `compact`/`standard`.
- Бандлинг самого шрифта JetBrains Mono (по умолчанию системный mono-фолбэк).
- Горизонтальное масштабирование сервера (in-process lock достаточно для одного инстанса).
